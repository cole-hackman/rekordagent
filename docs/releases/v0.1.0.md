# v0.1.0 — Foundations & Read-Only Library Browser

_Phase 1 complete._

## What ships

### Library browser
- Virtualized track table (TanStack Table + Virtual) handles 50k+ tracks without frame drops.
- Sort by any column; filter by title, artist, album, genre, or comment.
- Track detail panel: full metadata, star rating, year, play count, comment.

### Audio preview
- Click any track to load it in the detail panel; click the play button or press **Space** to preview.
- Playback handled by `rodio` on a dedicated OS thread — no latency spikes.

### Waveform display
- WaveSurfer.js renders the audio waveform for the selected track via the Tauri `asset://` protocol.
- Hot cue positions overlaid as yellow markers.
- Play/pause cursor synced to the rodio player.

### Hot cues
- All cue types (memory cue, hot cues 1–8) displayed with Rekordbox colours and millisecond timestamps.

### Settings
- Dark / light theme toggle, persisted to disk.
- Library path change with file-picker validation.
- Anthropic API key stored via OS keychain (`keyring` crate).

### Agent chat
- Streaming Claude chat panel (toggle with the speech-bubble button).
- Auto-detects the `claude` CLI and uses your subscription without requiring a separate API key.
- Falls back to a manually entered Anthropic API key if the CLI is not found.
- Tools: `library.search`, `library.list_playlists`, `health.orphan_scan`.

### Foundation crates
- `rekordbox-db` — read-only SQLCipher access; tracks, playlists, hot cues, beat grid (ANLZ).
- `rekordbox-xml` — Rekordbox XML parse + emit with round-trip property tests.
- `cache` — SQLite WAL store with schema versioning.

## Known limitations (Phase 2 targets)
- No waveform scrubbing (click-to-seek on the waveform doesn't yet move the playhead).
- No playlist browser panel in the UI.
- Agent tool set is minimal (3 tools); full ~50-tool inventory comes in v0.2.0.
- No staged change / diff workflow yet.
